# Email Integration für JARVIS\n\n## Microsoft Outlook / Office365 Email-Integration\n\nJARVIS unterstützt Microsoft-verwaltete E-Mails über IMAP/SMTP mit App-Passwörtern.\n\n### Konfiguration\n\n1. **Office365 App-Passwort erstellen**\n   - Gehe zu: https://account.microsoft.com/account/manage-my-microsoft-account\n   - \"Sicherheit\" → \"App-Passwörter\"\n   - Erstelle ein neues App-Passwort für \"Mail\"\n   - Kopiere das 16-stellige Passwort\n\n2. **In `.env` eintragen:**\n```\nEMAIL_PROVIDER=microsoft\nEMAIL_ADDRESS=your-email@company.onmicrosoft.com\nOFFICE365_IMAP_HOST=imap.office365.com\nOFFICE365_IMAP_PORT=993\nOFFICE365_SMTP_HOST=smtp.office365.com\nOFFICE365_SMTP_PORT=587\nOFFICE365_APP_PASSWORD=xxxx-xxxx-xxxx-xxxx  # 16-stelliges App-Passwort\n```\n\n### API Endpoints\n\n#### 1. Test Email Connection\n```bash\nPOST /api/v1/email/test-connection\n```\n\nBeispiel:\n```bash\ncurl -X POST http://localhost:8000/api/v1/email/test-connection\n```\n\nResponse:\n```json\n{\n  \"status\": \"success\",\n  \"smtp\": \"✓ Connected\",\n  \"imap\": \"✓ Connected\",\n  \"email\": \"your-email@company.onmicrosoft.com\"\n}\n```\n\n#### 2. Get Unread Emails mit AI-Response Suggestions\n```bash\nGET /api/v1/email/unread?limit=20\n```\n\nBeispiel:\n```bash\ncurl http://localhost:8000/api/v1/email/unread\n```\n\nResponse:\n```json\n{\n  \"count\": 2,\n  \"emails\": [\n    {\n      \"id\": 123,\n      \"from\": \"boss@company.com\",\n      \"to\": \"you@company.com\",\n      \"subject\": \"Project Update\",\n      \"body\": \"Can you provide status on the Q2 project?\",\n      \"received_at\": \"2026-06-17T10:30:00\",\n      \"ai_response_suggestion\": \"I'll have the Q2 project status ready by end of day. Currently on track with initial milestones. Will send detailed report shortly.\"\n    }\n  ]\n}\n```\n\n#### 3. Send Email\n```bash\nPOST /api/v1/email/send\n```\n\nBody:\n```json\n{\n  \"to_address\": \"recipient@company.com\",\n  \"subject\": \"Project Status\",\n  \"body\": \"Here's the Q2 project status...\",\n  \"html_body\": \"<p>Here's the Q2 project status...</p>\",\n  \"cc\": [\"manager@company.com\"],\n  \"bcc\": []\n}\n```\n\n#### 4. Mark Emails as Read\n```bash\nPOST /api/v1/email/mark-as-read\n```\n\nBody:\n```json\n{\n  \"email_ids\": [123, 124, 125]\n}\n```\n\n#### 5. Email System Status\n```bash\nGET /api/v1/email/status\n```\n\n### Features\n\n✅ **AI Response Suggestions**\n- Automatische Generierung von professionellen Email-Antworten\n- Basierend auf lokales Ollama LLM (Mistral)\n- Nur lokal - keine Daten nach außen\n\n✅ **Email Management**\n- Unread Emails abrufen\n- Emails als gelesen markieren\n- Neue Emails senden (mit CC/BCC)\n\n✅ **Integration**\n- IMAP für Email-Abruf\n- SMTP für Email-Versand\n- OAuth2-ready (Phase 2)\n\n✅ **Security**\n- App-Passwort verschlüsselt gespeichert\n- Nur lokale Verarbeitung\n- Keine Credentials in Logs\n\n### Troubleshooting\n\n**\"Authentifizierung fehlgeschlagen\"**\n- Überprüfe ob App-Passwort (16 Zeichen) verwendet wird\n- Nicht das reguläre Passwort nutzen\n- IMAP/SMTP in Mailbox aktiviert?\n\n**\"Verbindung abgelehnt\"**\n- Firewallregeln überprüfen\n- SMTP Port 587 (TLS) testen\n- IMAP Port 993 (SSL) testen\n\n**\"Keine Emails gefunden\"**\n- Unread Emails vorhanden?\n- Locale korrigieren für Inbox-Name\n\n### Phase 2: OAuth2 Integration\n\nGeplant für bessere Sicherheit:\n- OAuth2 für Authentifizierung\n- Keine Passwords nötig\n- Automatisches Token Refresh\n- Multi-Account Support\n
